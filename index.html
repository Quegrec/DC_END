04 · Construire · Bloc DEV · Justification de la stack

Chaque choix répond à un besoin métier

Next.js 15 // frontend hybride

SSR sur les pages de guides → indexation SEO optimale (canal d’acquisition de Camille). CSR sur le dashboard → interactivité fluide sans rechargement (exigence d’Alex).

Symfony 7 + API Platform // backend

Sécurité native (XSS, CSRF, CSP), rigueur par injection de dépendances. API Platform expose Guides & Archétypes en JSON-LD, interopérable avec des API tierces (Steam, Riot).

PostgreSQL + JSONB // persistance

Relationnel pour comptes et rôles ; JSONB binaire indexé pour les scores d’archétypes évolutifs → l’algorithme itère sans migration lourde du schéma.

API IGDB // données externes

Index mondial des métadonnées de jeux : on ne reconstruit pas l’encyclopédie, on la consomme et on la met en cache (Redis) pour maîtriser latence et quotas.

  • Principe : un choix techno se justifie par un besoin métier traçable, jamais par la mode
  • Next.js 15 : SSR sur les guides → indexation SEO (canal d'acquisition de Camille) · CSR sur le dashboard → fluidité sans rechargement (exigence d'Alex)
  • Symfony 7 : sécurité native (XSS, CSRF, CSP), injection de dépendances → code testable · API Platform : JSON-LD standard, interopérable (Steam/Riot demain)
  • PostgreSQL JSONB : scores d'archétypes évolutifs stockés en binaire indexé → l'algo itère sans migration de schéma ; relationnel pour les comptes
  • IGDB + cache Redis : latence et quotas maîtrisés

Ce que le marathon m’a appris

Posture Tech Lead

Arbitrer, c’est renoncer : périmètre MVP réduit à 3 genres, IA complexe reportée, fonctionnalités hors périmètre assumées. La valeur avant la prouesse.

Management

Gouvernance à 3 niveaux (Daily, Sprint Review, COPIL), matrice RACI, règle 1 anomalie = 1 ticket : la clarté des rôles libère l’équipe, elle ne la contraint pas.

Gestion de l’incertitude

V1 → V2 du prototype : onboarding raccourci, hiérarchie visuelle refondue, contrastes WCAG. Le feedback utilisateur tranche là où l’intuition débat.

De l’idéation à la viabilité financière, GGHF démontre qu’une conviction de joueur peut devenir un produit architecturé et un modèle économique crédible.

  • Management : gouvernance 3 niveaux (daily · sprint review + panel · COPIL mensuel) + RACI + 1 anomalie = 1 ticket → la clarté des rôles libère l'équipe
